## 1. 项目概述 (Overview)

### 1.1 产品定位
一款专为“任务驱动型消费”设计的个人管理工具。通过游戏化机制，将日常任务转化为虚拟奖金，并严格核销特定类型的消费。产品强调**隐私安全**（数据本地化）、**极致便捷**（语音交互/快捷入口）和**灵活智能**（可配置 AI 分类）。

### 1.2 核心价值
*   **双轨制管理**：清晰区分“奖金池收支”与“日常流水”，避免财务混淆。
*   **零摩擦记录**：通过桌面快捷方式和语音 AI，实现秒级记账。
*   **数据主权**：用户完全掌控数据，无云端依赖，支持手动备份。

## 3. 功能需求详解 (Functional Requirements)

### 3.1 核心业务逻辑：双账户体系

#### 3.1.1 任务模块 (Tasks)
*   **任务属性**：
    *   `名称` (String)
    *   `奖金金额` (Decimal, 可为 0；若类型为 NORMAL_TASK，该字段无效但可设置为 0)
    *   `类型` (Enum: `REWARD_TASK` / `NORMAL_TASK`)
    *   `连续目标` (Int, 可选，v1.1 启用；v1.0 预留字段，前端不可见，代码中添加 TODO 注释)
    *   `当前连续天数` (Int)
    *   `历史最大连续` (Int)
    *   `打卡记录` (List of CheckinRecord，每个记录包含：`日期`、`状态`：正常/逾期/未打卡)
*   **打卡逻辑**：
    *   每天最多打卡一次同一任务（按自然日 00:00-23:59 判断）。
    *   用户点击打卡按钮，记录当天为“正常完成”，更新当前连续天数（若前一天有打卡则 +1，否则重置为 1）。
    *   **补卡规则**：允许用户补打卡过去 **3 天**（含当天）内未打卡的日期。补卡时，用户选择具体日期（仅可选未打卡的日期），标记为“逾期完成”。补卡后连续天数计算规则：补卡当天视为完成，连续天数按实际完成日是否连续计算（即如果用户之前连续 N 天，补卡某缺失日，则从该缺失日到当前日若连续，则连续天数 = 缺失日到当前日的天数，否则重置）。详细计算由代码实现，确保统计准确。
    *   若当天未打卡且未在后续 3 天内补卡，则该天状态自动标记为“未打卡”，连续天数归零。
    *   打卡后奖金发放：若任务类型为 `REWARD_TASK`，每次正常打卡或逾期打卡均增加 `奖金金额` 到余额（v1.0 暂不区分正常/逾期，v1.1 可配置奖惩差异）。
*   **CRUD 操作**（增删改查）：
    *   用户可创建、编辑、删除任务。
    *   删除任务时需确认，并提示相关打卡记录也将被删除。

#### 3.1.2 账单模块 (Transactions)
*   **账单属性**：
    *   `金额` (Decimal)
    *   `类别` (String，从预设分类中选择，AI 生成后用户可修改)
    *   `描述` (String, 原始语音/文本)
    *   `是否核销奖金` (Boolean: `REIMBURSABLE` / `PERSONAL`)
    *   `时间` (Timestamp)
    *   `AI 置信度` (Float, 可选)
*   **执行逻辑**：
    *   若 `是否核销奖金 == true` -> `总余额 -= 金额`。
    *   若 `是否核销奖金 == false` -> 仅记录流水，余额不变。
*   **手动添加入口**：主界面提供“手动记账”按钮，可填写所有字段，支持从预设分类中选择。
*   **编辑功能**：用户可修改任意已保存账单的所有字段。

### 3.2 智能 AI 交互模块

*   **输入**：用户语音转写的文本 (e.g., "刚花了 35 买咖啡，算奖金里")。
*   **处理流程**：
    1.  读取用户配置的 API Key、Base URL 和 Model Name。
    2.  发送 Prompt 给大模型，Prompt 中包含预定义的分类列表，要求模型从中选择最匹配的分类。
    3.  解析 JSON 响应。
*   **配置项 (设置页)**：
    *   `API Key` (加密存储)
    *   `API Base URL` (例如：`https://api.openai.com/v1`)
    *   `Model Name` (例如：`gpt-3.5-turbo`，默认可留空，由服务端决定)
    *   `AI 确认模式` (Enum):
        *   `总是确认`：每次语音后都弹窗预览，用户需手动点“确认”才能保存。
        *   `智能确认`：仅当 AI 返回 `confidence < 0.8` 或识别失败时弹窗；否则直接保存。
        *   `静默模式`：永远不弹窗，直接保存。若 AI 置信度过低（<0.5），仍直接保存，但在账单列表中该条目显示“需复核”标记，点击可快速修改。
*   **预设分类**：
    *   默认分类：`餐饮`、`交通`、`购物`、`娱乐`、`医疗`、`教育`、`家居`、`人情`、`其他`
    *   用户可在设置中自定义分类列表（v1.1 或 v1.2 支持）。
*   **异常处理**：
    *   若 AI 调用失败 (网络错误/Key 无效) -> 降级为“手动输入模式”，保留文字让用户自己填金额和分类。
    *   无网络时，语音入口禁用，提示“需要网络连接”，用户只能使用手动记账。

### 3.3 统计与分析模块

*   **Tab 1: 奖金银行 (Bonus Bank)**
    *   **数据源**：仅包含 `REWARD_TASK` 的收入 和 `REIMBURSABLE=true` 的支出。
    *   **图表**：
        *   本月净收益 (柱状图)。
        *   奖金消耗分类占比 (饼图，基于 `REIMBURSABLE` 支出)。
        *   当前可用余额 (大字卡片，余额允许负数，负数红色显示)。
*   **Tab 2: 任务与生活 (Tasks & Life)**
    *   **数据源**：所有任务 + 所有账单。
    *   **图表**：
        *   任务完成率趋势 (折线图，每日完成数/总任务数)。
        *   连续打卡天数排行 (条形图，展示各任务当前连续天数)。
        *   生活总支出趋势 (无论是否核销)。

### 3.4 系统与安全模块

*   **数据存储**：使用 **Isar** 数据库，本地存储（不加密，因数据完全本地且无敏感信息）。API Key 使用 `flutter_secure_storage` 加密存储。
*   **备份机制**：
    *   **导出**：调用系统分享接口，生成 Isar 数据库文件 (`.isar`) 或 JSON 备份。
    *   **导入**：选择备份文件 -> 自动备份当前数据（生成时间戳副本） -> 校验格式 -> 覆盖本地数据库 -> 提示用户重启应用。
*   **无网络处理**：语音功能在无网络时不可用（按钮置灰或隐藏），提示用户开启网络或使用手动记账。本地 ASR 不启用，避免复杂度。

## 4. 非功能需求 (Non-Functional Requirements)

*   **性能**：
    *   App 冷启动时间 < 1.5 秒。
    *   语音转文字 + AI 解析全流程目标在 3 秒内完成（取决于网络状况）。
    *   列表滑动帧率 > 50fps。
*   **兼容性**：
    *   最低支持 Android 10.0。
    *   适配主流屏幕分辨率。
*   **可靠性**：
    *   无网络环境下，手动记账功能完全可用。
    *   数据写入具备事务性，防止断电导致数据损坏。
*   **安全性**：
    *   不收集任何用户个人信息上传服务器。
    *   API Key 仅存于本地加密存储。

## 6. 附录：AI Prompt 策略草案 (供开发参考)

*系统将在后台使用类似以下的 Prompt 模板：*

```text
你是一个个人财务助手。请从用户的输入中提取信息，并从给定的分类列表中选择最合适的一项，返回严格的 JSON 格式。
用户输入: "{user_text}"
当前配置: {allow_reimburse: true/false}
可选分类: ["餐饮", "交通", "购物", "娱乐", "医疗", "教育", "家居", "人情", "其他"]

请返回:
{
  "amount": float, // 金额，若未提及则为 0
  "category": string, // 必须从可选分类中选择
  "is_reimbursable": boolean, // 是否建议从奖金扣除 (根据语境判断，如"帮朋友付"则为 false)
  "confidence": float, // 0.0-1.0 置信度
  "summary": string // 简短摘要
}
注意：如果用户明确说"不算奖金"，则 is_reimbursable 必须为 false。
```
